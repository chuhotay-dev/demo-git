*Chú thích - wd: working directory - sa: staging area - lp: local repository 1. git init : Tạo ra local repo (.git) 2. git status : kiểm tra xem những thay đổi của code (biết được file đang nằm ở vùng nào) 3. git add [ten_file] : chuyển 1 file từ vùng wd sang vùng sa 4. git add . : chuyển tất cả file (có sự thay đổi) từ vùng wd sang vùng sa 5. git rm --cached [ten_file] : đưa một file từ vùng sa quay ngược lại vùng wd 6. git commit -m "your message" : đưa tất cả file có sự thay đổi từ vùng sa sang lp, bắt buộc phải có message 7. config git acc + git config --global user.name "Your Name" + git config --global user.email you@example.com 8. git log:  kiểm tra lịch sử commit 9. git remote add origin [remote_repo_url] : tạo kết nối giữa lp với remote repo 10. git push origin [ten_branch]: đẩy code dưới lp lên remote repo (áp dụng trong trường hợp ten_branch chưa có ở remote repo) 11. git clone [url_onlone_repo] : lấy src code từ github về máy 12, git checkout -b [ten_branch] : tạo ra một branch mới và chuyển qua branch vừa tạo 13. git pull: kéo code mới nhất từ remote repo 14: git checkout [ten_branch] : chuyển branch hiện tại qua ten_branch 15. git branch : kiểm tra xem đang ở nhánh nào
